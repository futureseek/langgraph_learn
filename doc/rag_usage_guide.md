# RAG功能使用指南

## 什么是RAG？

RAG（Retrieval-Augmented Generation，检索增强生成）是一种结合了信息检索和自然语言生成的AI技术。它通过以下步骤工作：

1. **文档索引**：将文档分块并转换为向量存储
2. **检索相关内容**：根据用户问题检索最相关的文档片段
3. **生成回答**：基于检索到的内容生成准确回答

## 如何使用RAG功能

### 1. 添加文档到知识库

你可以通过以下方式添加文档：

```
用户: 请将 README.md 添加到RAG知识库
```

系统会自动调用 `add_document_to_rag` 工具处理文档。

### 2. 批量添加文档

```
用户: 将 docs 目录下的所有文档添加到知识库
```

### 3. 基于知识库问答

```
用户: 根据知识库回答：这个系统支持哪些文件格式？
```

## 支持的文件格式

RAG系统支持以下文件格式：
- 文本文件：.txt, .md, .log
- 代码文件：.py, .js, .java, .cpp, .c, .h
- 数据文件：.json, .xml, .csv
- 文档文件：.pdf, .docx
- 配置文件：.cfg, .ini, .yaml, .yml

## 高级功能

### 知识库管理

- 查看知识库状态：`get_rag_stats()`
- 删除文档：`delete_rag_document(文件路径)`
- 清空知识库：`clear_rag_knowledge_base()`

### 技术特性

- **智能分块**：自动将长文档分割为合适的块
- **语义检索**：使用向量相似度进行精确检索
- **上下文增强**：将检索结果作为上下文提供给LLM
- **来源追踪**：回答中包含信息来源

## 实际应用场景

1. **项目文档问答**：为项目文档建立智能问答系统
2. **技术文档检索**：快速查找技术资料中的特定信息
3. **知识管理**：构建个人或团队知识库
4. **客服系统**：基于FAQ和手册的自动回答

## 注意事项

- 文档质量影响回答质量
- 建议定期更新知识库内容
- 大文件会自动分块处理
- 支持中英文混合文档
